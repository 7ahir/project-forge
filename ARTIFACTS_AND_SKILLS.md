# Artifacts and Skills Map

## Why This File Exists

The main portfolio docs are designed to foreground choices, tradeoffs, and product judgment.

That is good for recruiter readability, but it can hide some of the PM craft underneath the work. This file makes the underlying skillset explicit so a reviewer can answer three questions quickly:

1. What PM frameworks are being used?
2. Which artifacts prove which PM skills?
3. Does this project actually demonstrate product management craft, or just polished writing?

## What I Deliberately Did Not Do

- I did not make every file read like a framework checklist.
- I did not name-drop methodology in place of argument quality.
- I did not optimize the project for PM jargon density.

The goal is a stronger balance:

- visible product judgment in the main narrative
- explicit PM craft in this map and in each artifact's skill callout
